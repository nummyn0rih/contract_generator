# Contract Generator

Автогенерация договоров с фермерами на основе шаблона `.docx` и базы фермеров `.xlsx/.csv`.
Индивидуальные пункты и финальная проверка — через GigaChat API (Sber).

## Возможности

- Массовая генерация пакета договоров одной кнопкой.
- Pydantic-валидация ИНН, КПП, БИК и расчётного счёта по алгоритмам ЦБ/ФНС.
- AI-генерация индивидуальных пунктов по карточке фермера (культура, регион, объём).
- AI-проверка готового договора по чек-листу: пустые плейсхолдеры, рассогласованность сумм, реквизитов, дат.
- Скачивание всего пакета одним ZIP-архивом.
- Лог по каждому фермеру (loguru).

## Запуск

```bash
python -m venv .venv
source .venv/bin/activate            # Windows: .venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env                 # вписать GIGACHAT_CREDENTIALS

streamlit run app.py
```

Откроется браузер на `http://localhost:8501`.

### Получение GIGACHAT_CREDENTIALS

1. Зайти в [Sber Studio](https://developers.sber.ru/studio), залогиниться через Sber ID.
2. Создать проект GigaChat API, скопировать **Authorization Key** (Base64-токен).
3. Вставить значение в `.env` как `GIGACHAT_CREDENTIALS=...`.

По умолчанию `GIGACHAT_VERIFY_SSL=false` — работает без установки корневого сертификата
Минцифры. Для прода рекомендуется поставить корневой сертификат и переключить флаг в `true`
(см. [документацию SaluteDevs](https://developers.sber.ru/docs/ru/gigachat/certificates)).

## Использование

1. В UI можно либо загрузить свой `template.docx` и `farmers.xlsx`, либо нажать «Использовать пример» — подгрузятся файлы из `data/`.
2. В превью видны фермеры с цветовой маркировкой:
   - зелёный — данные валидны;
   - красный — есть ошибки в реквизитах (фермер пропускается).
3. Кнопка «Сгенерировать пакет» запускает пайплайн: AI-генерация пунктов → заполнение шаблона → AI-проверка готового документа.
4. Готовые `.docx` лежат в `output/` и доступны для скачивания (поштучно или ZIP).

## Шаблон договора

В `template.docx` поддерживаются следующие плейсхолдеры:

| Плейсхолдер                  | Значение                                  |
| ---------------------------- | ----------------------------------------- |
| `{{ФИО}}`                    | ФИО фермера                               |
| `{{ИНН}}`                    | ИНН (10 или 12 цифр)                      |
| `{{КПП}}`                    | КПП (для юрлиц)                           |
| `{{БАНК}}`                   | Название банка                            |
| `{{БИК}}`                    | БИК                                       |
| `{{Р_СЧЕТ}}`                 | Расчётный счёт                            |
| `{{К_СЧЕТ}}`                 | Корреспондентский счёт                    |
| `{{РЕГИОН}}`                 | Регион                                    |
| `{{КУЛЬТУРА}}`               | Культура                                  |
| `{{ОБЪЕМ}}`                  | Объём, тонн                               |
| `{{ЦЕНА_ЗА_ТОННУ}}`          | Цена за тонну, руб.                       |
| `{{СУММА}}`                  | Итоговая сумма, руб.                      |
| `{{СУММА_ПРОПИСЬЮ}}`         | Та же сумма прописью                      |
| `{{ДАТА}}`                   | Дата договора                             |
| `{{СЕЗОН}}`                  | Сезонный год                              |
| `{{ИНДИВИДУАЛЬНЫЕ_УСЛОВИЯ}}` | AI-сгенерированный блок индивидуальных пунктов |

Word может разбивать плейсхолдер на несколько runs — `docx_filler` это учитывает.

## Структура

```
contract_generator/
├── app.py              # Streamlit UI
├── core/               # бизнес-логика
├── prompts/            # системные/юзер промпты для GigaChat — редактируйте здесь
├── data/               # пример шаблона и базы фермеров
├── output/             # готовые договоры (gitignored)
├── logs/               # loguru-логи
└── requirements.txt
```

Все промпты к LLM — в `prompts/*.md`. Можно править без перезапуска приложения.

## Ограничения и планы

- AI-сгенерированные пункты и AI-валидация **не заменяют юриста**. Это инструмент сокращения рутины, а не источник истины.
- Текущий шаблон `data/template.docx` — заглушка для демо. Замените его реальным договором, сохранив (или расширив) перечень плейсхолдеров.
- Roadmap: тюнинг промптов под реальный договор → деплой → интеграция с 1С/CRM.
